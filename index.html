Cloud Computing

Historia

En la actualidad hablamos del almacenamiento en la nube como algo muy de ahora, del futuro, y lo cierto es que el origen de esta tecnología tiene lugar desde la década de los 50's - 60's. Aunque muy pocas personas lo sepan, los primeros avances del cloud computing se dieron a partir de los años 50, cuando se torno como necesidad el uso de ordenadores para poder almacenar información. Desde ese entonces, han habido una serie de avances que nos han llevado a que actualmente usemos la nube como método de almacenamiento de datos.

A John McCarthy se le atribuye este concepto en 1961, a quien tambien se le atribuye el concepto de inteligencia artificial. Pero otros autores le atribuyen el concepto de cloud computing a Joseph Carl Robnett Licklider, quien pretendía que todo el mundo pudiese estar interconectado para acceder a los programas y datos desde cualquier lugar y dispositivo.

Punto de vista del usuario

Desde el punto de vista del usuario, los aspectos que caracterizan a la experiencia de cloud computing son:

  • En lugar de tener que comprar, instalar, ejecutar y mantener una aplicación de software localmente, se utiliza un navegador para conectarse a una aplicación que el proveedor de servicios en la nube posee y mantiene, y para utilizar esa aplicación.
  • El usuario debe pagar por el servicio en función de la cantidad del mismo que utilice y del tiempo durante el que lo utilice.

Punto de vista del proveedor

Para el proveedor de servicios de cloud computing, la nube se define por su aprovechamiento de las mejores y más recientes tecnologías de centros de datos. En concreto, cloud computing requiere un amplio uso de la virtualización (servidores, almacenamiento y red) y de la automatización (en particular del aprovisionamiento, el desaprovisionamiento, la escalabilidad y la migración de cargas de trabajo). Cloud computing también se define por la compatibilidad con varios inquilinos de manera segura y por la implantación de API orientadas a la Web (como las API HTTP RESTful) que facilitan el autoservicio de los clientes a través de portales web o aplicaciones cliente creadas por los clientes.

Modelos de despliegue

Exisaten 3 díiferentes modelos en los que podemosutilizar la nube estos modelos son:

  • Privada:

    La nube privada, es un concepto el cual se utiliza para servidores que trabajan usual mente con información empresarial de nivel de servicio, que debe de ser administrada por uno o varios usuarios con alto conocimiento en el tema. Tienen un alto nivel de seguridad y Firewalls así protegen la intgridad de los datos, normalmente para este tipo de nube se necesita mas recursos y presupuesto.

  • Pública:

    Son administradas por terceros y la informacion de diferentes personas se ubican mezcladas en los mismos sistemas de almacenamiento, la nube no diferencia los dato un de otro y los corre al mismo tiempo lo que la hace mas versatil y menos costosas.

  • Híbrida:

    Es un nuevo tipo de computación que utiliza la nube privada para ejecutar sus procesos más importantes y una nube publica par el resto de tareas, una API se encarga de comunicar y poner asignar las tareas de ambas nubes.

=======

El cloud computing es un sistema de almacenamiento de datos que permite a las empresas guardar sus archivos en servidores y acceder a ella desde cualquier parte del mundo a través de un dispositivo con conexión a Internet. El almacenamiento en la nube permite a los usuarios utilizar y acceder de forma remota a software y hardware a través de Internet.

El concepto de «nube» es muy amplio, y abarca casi todos los posibles tipo de servicio en línea, pero cuando las empresas ofrecen un espacio alojado en la nube, es especialmente el almacenamiento de datos y hardware (capacidad de cómputo). El término se utiliza generalmente para describir los servidores de datos disponibles desde cualquier lugar del mundo, para muchos usuarios a través de Internet desde cualquier dispositivo.

Tipos de servicios en la nube

Otra parte útil de una definición de cloud computing es una categorización de los tipos de servicios en la nube.

  • Infraestructura como servicio (IaaS):

    Capacidad primaria de procesamiento y almacenamiento en ordenadores, el cual es un metodo donde las empresas de centro de datos que optan para facilitar la infraestructura de sofwer al usuario y desde el punto de vista del usuario avaratar costos. En el I.A.A.S el usuario que contrta el servcio de infraestructura tiene las opciones de elegir cuanta capacida de procesamiento nececita, memoria ram, procesamiento grafico, todo esto desde una maquina virtal la cual esta sostenida en una o varias comutadoras con la funcion de cervidores.

  • Plataforma como servicio (PaaS):

    Plataformas de desarrollo en las que los programadores pueden crear y probar nuevas aplicaciones.

  • Software como servicio (SaaS): Aplicaciones de software preparadas para una amplia variedad de necesidades de consumidores o empresas.
  • Plataforma como servicio (PaaS): Plataformas de desarrollo en las que los programadores pueden crear y probar nuevas aplicaciones.
