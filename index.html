Cloud Computing

Historia

En la actualidad hablamos del almacenamiento en la nube como algo muy de ahora, del futuro, y lo cierto es que el origen de esta tecnología tiene lugar desde la década de los 50's - 60's. Aunque muy pocas personas lo sepan, los primeros avances del cloud computing se dieron a partir de los años 50, cuando se torno como necesidad el uso de ordenadores para poder almacenar información. Desde ese entonces, han habido una serie de avances que nos han llevado a que actualmente usemos la nube como método de almacenamiento de datos.

A John McCarthy se le atribuye este concepto en 1961, a quien tambien se le atribuye el concepto de inteligencia artificial. Pero otros autores le atribuyen el concepto de cloud computing a Joseph Carl Robnett Licklider, quien pretendía que todo el mundo pudiese estar interconectado para acceder a los programas y datos desde cualquier lugar y dispositivo.

Definición

Consiste en la optención de diferentes tipos de servicios de manera temporal o permanente estos servicios permiten a los usuarios utilizar y acceder de forma remota a hardware y software informático a través de Internet.

Punto de vista del usuario

Desde el punto de vista del usuario, los aspectos que caracterizan a la experiencia de cloud computing son:

  • En lugar de tener que comprar, instalar, ejecutar y mantener una aplicación de software localmente, se utiliza un navegador para conectarse a una aplicación que el proveedor de servicios en la nube posee y mantiene, y para utilizar esa aplicación.
  • El usuario debe pagar por el servicio en función de la cantidad del mismo que utilice y del tiempo durante el que lo utilice.

Punto de vista del proveedor

Para el proveedor de servicios de cloud computing, la nube se define por su aprovechamiento de las mejores y más recientes tecnologías de centros de datos. En concreto, cloud computing requiere un amplio uso de la virtualización (servidores, almacenamiento y red) y de la automatización (en particular del aprovisionamiento, el desaprovisionamiento, la escalabilidad y la migración de cargas de trabajo). Cloud computing también se define por la compatibilidad con varios inquilinos de manera segura y por la implantación de API orientadas a la Web (como las API HTTP RESTful) que facilitan el autoservicio de los clientes a través de portales web o aplicaciones cliente creadas por los clientes.

Modelos de despliegue

Exisaten 3 díiferentes modelos en los que podemosutilizar la nube estos modelos son:

  • Privada:

    La nube privada, es un concepto el cual se utiliza para servidores que trabajan usual mente con información empresarial de nivel de servicio, que debe de ser administrada por uno o varios usuarios con alto conocimiento en el tema. Tienen un alto nivel de seguridad y Firewalls así protegen la intgridad de los datos, normalmente para este tipo de nube se necesita mas recursos y presupuesto. Parte de sus desventajas es que el mantenimiento de la nube queda a cargo de la propia empresa y esto depende de como se gestione puede favorecer o afectar a la empresa.

  • Pública:

    Son administradas por terceros y la informacion de diferentes personas se ubican mezcladas en los mismos sistemas de almacenamiento, la nube no diferencia los dato un de otro y los corre al mismo tiempo lo que la hace mas versatil y menos costosas. A pesar de no ser tan segura la nube pública presenta un gran punto a favor la escalabilidad de los datos es mucho mas rápida y eficiente y el mantenimiento es gestionado por el proveedor los cuales normalmente son bastante competentes.

  • Híbrida:

    Es un nuevo tipo de computación que utiliza la nube privada para ejecutar sus procesos más importantes y una nube publica par el resto de tareas, una API se encarga de comunicar y poner asignar las tareas de ambas nubes.La flexibilidad es una caracteristica importante de las híbridas ya que podemos combinar diferentes tipos de servicios e infraestructura segun lo requiramos sin embargo se pueden presentar problemas de compatibilidad y conectividad si no se gestiona de buena manera.

Tipos de servicios en la nube

Otra parte útil de una definición de cloud computing es una categorización de los tipos de servicios en la nube.

  • Infraestructura como servicio (IaaS):

    Capacidad primaria de procesamiento y almacenamiento en ordenadores, el cual es un metodo donde las empresas de centro de datos que optan para facilitar la infraestructura de sofwer al usuario y desde el punto de vista del usuario avaratar costos. En el I.A.A.S el usuario que contrata el servcio de infraestructura tiene las opciones de elegir cuanta capacida de procesamiento nececita, memoria ram, procesamiento grafico, alamacenamiento, todo esto desde una maquina virtal la cual esta sostenida en una o varias comutadoras con la función de servidores.

  • Plataforma como servicio (PaaS):

    Posee todas las caracteristicas del Iaas pero con la opción de programs aplicaciones sencillas o complejas basadas totalmente en la nube adquiriendo los recursos necesarios mediante la misma nube y solo se paga por el servicio solicitado y usado .

  • Software como servicio (SaaS):

    Posee todas las caracteristicas de PaaS pero tiene la ventaja de tener acceso desde cualquier lugar con internet y ejecutar la aplicación desde cualquier navegador ademas de ejecutarse completamente en el servior, no se debe pagar por el acceso de un usuario a la aplicación.

Principales provedores del Cloud Computing

  1. Microsoft.

    Está en el puesto número uno por varios motivos. Ofrece servicios en las tres capas de la nube (IaaS, PaaS y SaaS). Está comprometido en ayudar en el desarrollo y la implementación de sus clientes en entornos de producción innovadores. Tiene mas de 100 servicios integrados, tiene a su disposicion servicios de inteligencia artificial para crear aplicaciones inteligentes y demas. Es un sistema muy veloz, tambien ofrece cobertura global y ademas de eso es eficientemente seguro.

  2. Amazon.

    Amazon Web Services esta en el segundo puesto por la profundidad y amplitud de productos y servicios que ofrece. Además, de la experiencia de usuario sencilla y elegante, ademas de una precios muy competitivos. Se encuentra en el número dos, por las carencias respecto a Microsoft en aspectos de software. AWS ofrece una excelente facilidad de uso, es muy flexible; permite seleccionar el sistema operativo, la plataforma de aplicaciones web, el lenguaje de programación, la base de datos, etc. Tambien ofrece una escalabilidad y alto desempeño con las herramientas: Auto Scaling y Elastic Load Balancing. Tambien es muy seguro al igual que Microsoft.

  3. IBM

    Ultimamente tuvo una gran transformación de su amplia gama de experiencia y tecnología de software desde el entorno real hasta la nube. Al igual que Microsoft, IBM es uno de los proveedores de Cloud Computing que ofrece los tres servicios: IaaS, PaaS y SaaS. Esto es muy importante porque ofrece a los clientes más opciones, una integración más fluida, tiene una eficiencia inigualable, una escalabilidad muy eficiente, gracias a los servicios que brinda y ofrece una mejor seguridad cibernética que las demas.
